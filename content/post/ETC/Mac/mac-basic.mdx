## Mac에서 윈도우 화면을 여러 창으로 띄어서 사용하는 경우

- 보통 `Command + Tap` 사용하면 다른 창을 불러올 수 있지만, 같은 프로그램일 경우 불가능하다.
- 이 경우 `Command + ₩`  Command 버튼 + 숫자 1 왼쪽의 버튼을 누르면 같은 프로그램의 다른 창이 열립니다.(보통 듀얼모니터 사용 시 매우 유용)

## 터미널을 자주 사용한다면 `Iterm2` 설치를 추천

- `Iterm2`와 `powerlevel10k`그리고 `oh my zsh`를 설치하면 명령어 사용이 무척 편해진다.

## 윈도우의 `delete` 를 사용하고 싶다면

- `fn + backspace(뒤로가기 버튼)` 를 입력하면 됩니다.

## 제거한 인터넷(크롬, 사파리) 창의 탭을 복구

- `command + shift + 't'` 을 누르면 됩니다.

## 경로 설정

```shell
.   현재 디렉토리
..  부모 디렉토리
/   최상위 root
~   홈(HOME, 메인)
```


## 파일 찾기

- 파일 이름이 'abcde.mp4' 일때 `find` 명령어를 이용하여 찾을 수 있습니다.
- 소요 시간은 보통 1분 이상 걸리는것 같습니다.
- 또한, 아이클라우드 등에서도 검색을 피하고 싶다면 아래의 명령어를 이용하여 찾을 수 있습니다.

```bash
# 루트 디렉토리에서 시작해서 검색(오래 걸림)
find / -name abcde.mp4 2>/dev/null

# 홈디렉토리에서 시작해서 검색(덜 오래 걸림 - 그래도 오래 걸림)
find ~/ -name abcde.mp4 2>/dev/null

# 아이클라우드 등을 피하고 싶다면 아래의 명령어 이용
find / -path ~/Library/Mobile\ Documents -prune -o -name abcde.mp4 -print 2>/dev/null

```

- `path ~/Library/Mobile\ Documents`는 ~/Library/Mobile Documents 디렉토리를 찾습니다.
- `prune`은 해당 디렉토리를 검색 결과에서 제외합니다.
- o (or의 약자)는 뒤따르는 조건 (-name seq2seq.mp4)과 함께 사용되어, Mobile Documents 디렉토리를 제외한 나머지 경로에서 seq2seq.mp4 파일을 찾습니다.
- `print`는 찾은 파일의 경로를 출력합니다.
- `2>/dev/null`은 에러 메시지를 숨깁니다.
